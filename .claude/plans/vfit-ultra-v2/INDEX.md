## 📐 Regras do Plano

1. **Cada sprint é independente** — pode ser executado sem os seguintes
2. **Commits atômicos** — cada task = 1 commit com mensagem semântica
3. **Zero breaking changes** — B2B (personal) continua funcionando
4. **Mobile-first** — toda decisão prioriza a experiência mobile B2C
5. **TypeScript check** — zero erros após cada sprint
6. **Documentação inline** — header comment em todo arquivo tocado
