## B2C Onboarding Guard

**Problema resolvido:** alunos B2C (tipo `student`) que não completaram o onboarding conseguiam acessar o dashboard diretamente, pulando o fluxo de boas-vindas.

**Implementação:**

- `use-b2c-onboarding.ts` — hook que verifica se o onboarding foi concluído via localStorage e/ou API
- Guard no dashboard: se `effectiveType === 'student'` e `!completed` → `router.replace('/welcome')`
- `use-admin.ts` — ao mudar para modo `student`, remove a chave `vfit-onboarding` do localStorage para forçar re-execução do onboarding no contexto correto

**Fluxo:**
```
Login como student
       ↓
  Dashboard carrega
       ↓
 use-b2c-onboarding detecta onboarding incompleto
       ↓
  router.replace('/welcome')
       ↓
  Onboarding concluído → acesso ao dashboard liberado
```

## Fix crítico: `generateStaticParams` + `output: "export"`

**Root cause:** No Next.js 15.5.14, retornar `[]` (array vazio) de `generateStaticParams` é tratado como ausência da função, causando falha de build com `output: "export"`.

**Sintoma:** Build quebrava em rotas dinâmicas que retornavam array vazio, gerando erro de exportação estática.

**Fix aplicado:** Todas as rotas dinâmicas passaram a retornar `[{ id: '_' }]` — um placeholder não-vazio — garantindo que o Next.js reconheça a função e gere o arquivo estático correspondente.

**Padrão adotado nas rotas dinâmicas:**

```
app/dashboard/[id]/
  ├── page.tsx          ← Server Component com generateStaticParams
  └── client-page.tsx  ← Client Component com useParams()
```

**Fix adicional:** `useSearchParams()` encapsulado em `<Suspense>` onde necessário, evitando erro de hidratação.
